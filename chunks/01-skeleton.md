# Chunk 01 — Project Skeleton & Dev Environment

**Phase:** Foundations | **Estimated effort:** 2–3 hours | **Dependencies:** none

---

## Context

This chunk sets up the substrate the entire build runs on. Get it right and
every later chunk has a clean place to land. Get it wrong and you'll fight it
for weeks. Read `PROJECT.md` and `chunks/00-conventions.md` before starting.

This chunk produces no functional code. It produces the **shape of the
project** — directories, configuration, tooling, CI, and the commands that
let every later chunk run lint/type/test reliably.

---

## Scope

Create the repository skeleton, dev tooling, and minimum viable package layout
described in `PROJECT.md`. By the end of this chunk, `uv sync && pytest && ruff
check && mypy src/praxis` all pass on an empty-but-importable package.

### 4. `src/praxis/errors.py`

Define the full exception hierarchy from `00-conventions.md` §3:

```python
class PraxisError(Exception):
    def __init__(self, message: str, **details: object) -> None:
        super().__init__(message)
        self.details: dict[str, object] = dict(details)

class ConfigError(PraxisError): ...
class StorageError(PraxisError): ...
class TransportError(PraxisError): ...
class ToolError(PraxisError): ...
class SkillError(PraxisError): ...
class EngagementError(PraxisError): ...
class WorkqueueError(PraxisError): ...
class SufficiencyError(PraxisError): ...
class AuditError(PraxisError): ...
```

## Notes

- The package name on PyPI is `praxis-ba` (Python module is `praxis`) because
  `praxis` is taken on PyPI. The script entry point is still `praxis`.
- Use `uv` not `pip` everywhere. Document this in CONTRIBUTING.md.
- Keep the README short in this chunk — proper README is a later chunk.
- The `.praxis/` directory is in `.gitignore` because it's per-engagement
  runtime state, never checked in.
